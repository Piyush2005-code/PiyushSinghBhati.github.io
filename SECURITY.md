# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in the AI Robotics Visionary project, please **do not** open a public GitHub issue. Instead, please report it responsibly by:

1. **Email**: Send details to [security contact email]
2. **Subject**: Include `[SECURITY]` in the subject line
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce (if applicable)
   - Potential impact
   - Suggested fix (if any)

## Response Process

- We acknowledge receipt of vulnerability reports within 48 hours
- We work to verify and understand the scope of the issue
- We develop and test a fix
- We release a security patch and notify affected users
- We credit the reporter (unless they prefer anonymity)

## Security Best Practices for Users

When deploying this portfolio:

1. **Keep dependencies updated**: Regularly run `npm update` and review security advisories
2. **Use HTTPS**: Always deploy with HTTPS enabled
3. **Environment variables**: Never commit sensitive information
4. **Access control**: Use proper authentication if adding backend functionality
5. **CSP headers**: Configure Content Security Policy headers
